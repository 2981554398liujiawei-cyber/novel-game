# 对话立绘与演出规范

## 1. 目标

立绘用于快速确认说话人、强化情绪和增加陪伴感，不承担完整动画叙事。文本仍然是信息主载体；玩家即使关闭立绘或遇到缺图，也必须理解剧情、做出选择并完成游戏。

## 2. 节点行为

| 节点类型 | 默认立绘行为 |
|---|---|
| `dialogue` | 显示`speaker_id`对应立绘；使用`expression`，缺省时使用NPC默认表情 |
| `narrative` | 隐藏立绘；需要保持时使用`portrait_action: keep` |
| `choice` | 保持上一张立绘，直到玩家选择或后续节点改变 |
| `combat` | 隐藏大立绘，进入战斗UI |
| `reward` | 隐藏大立绘或保持上一个人物，由内容节点明确指定 |
| `complete` | 默认隐藏，章节告别场景可明确保持 |

允许的`portrait_action`为`show`、`keep`、`hide`。对白节点默认`show`，选择节点默认`keep`，其余默认`hide`。当前版本立绘位置固定在左侧，不开放每节点左右换位，避免布局和测试量膨胀。

## 3. 表情使用

- 只有人物圣经和NPC数据中登记的表情ID可使用。
- 同一段短对话不要频繁切换表情；一次情绪转折最多切换一次。
- 岚音`soft`只在高信任、明确放下防备的场景使用，不等于恋爱关系自动成立。
- 受伤表情只能在角色确实受伤或刚获救的节点使用；读档后应由当前节点重建正确状态。
- C级NPC只有`default`，不因一句嘲讽额外制作表情。

## 4. 姓名牌与对白

NPC姓名牌使用数据中的`name`。主角对白显示“枫月”，但当前版本不显示主角大立绘。系统文字不显示姓名牌；旁白可显示“旁白”或留空。对白正文不重复写“某某说道”来确认说话人，除非需要动作信息。

## 5. 转场与节奏

人物切换使用120—180毫秒淡入淡出；同一人物切换表情可直接交叉淡化。禁止闪白、镜头抖动、夸张缩放和长时间入场动画。玩家快速推进时转场可以立即完成，不能阻塞输入。

## 6. 降级与错误处理

- NPC数据缺少`portrait_set`：使用统一剪影。
- 指定`expression`不存在：使用该NPC默认表情并记录`PORTRAIT_EXPRESSION_MISSING`。
- 图片文件损坏：使用统一剪影并记录`PORTRAIT_ASSET_LOAD_FAILED`。
- 姓名、文本和选择必须继续显示；立绘错误不能导致主线卡死。

## 7. 验收场景

至少覆盖：同一NPC连续对白、两名NPC交替对白、从叙事进入对白、从对白进入选择、战斗前后恢复、快速跳过、自动播放、存档后读取、缺图、错误表情、720p、1080p和32px字号。

## 9. 无名对白角色

有正式对白但未具名的角色必须使用三套通用立绘之一：`GENERIC_VILLAGER_MALE`、`GENERIC_VILLAGER_FEMALE`、`GENERIC_MASKED_ADVENTURER`。纯背景人群不进入正式对话，不显示立绘。缺图剪影只用于资源异常，不能当作正式路人美术。
