# 战斗与检定完整规格

## 1. 定位

当前战斗难度为标准RPG：玩家需要观察敌人、搭配技能与消耗品，首次遭遇首领可能战败；理解机制并准备后应能稳定获胜。战斗不追求硬核刷数值，也不允许只按普通攻击无脑通过全部首领。

## 2. 基础回合

每回合流程：回合开始状态结算 → 按行动顺序执行单位行动 → 回合结束状态结算 → 检查胜负和阶段转换。单位行动顺序按`敏捷 + 本回合0—3随机值 + 状态修正`从高到低排列，随机值来自可保存的战斗种子；同值时玩家角色优先，其后按稳定ID排序。

玩家每回合选择六种基础动作：普通攻击、防御、技能、物品、观察、撤退。当前版本不设置法力值。技能使用冷却回合、每战次数或消耗品限制。

## 3. 六种动作

- 普通攻击：对单体造成武器伤害，可触发暴击和命中效果。
- 防御：从选择后持续到该角色下一次行动开始，所受直接伤害降低40%，不可叠加；特殊技能可提高到60%。
- 技能：技能最多装配4个，必须定义冷却、每战次数、目标、效果和失败处理。
- 物品：使用后消耗当前行动；治疗药不能超过生命上限。
- 观察：揭示敌人下一种危险动作、弱点或阶段规则；对已揭示目标可转为获得命中或防御加成。
- 撤退：进行撤退检定，成功进入任务续接节点，失败消耗行动。剧情锁定战斗必须提供明确的逃生条件，不能隐藏禁退。

## 4. 伤害与命中

- 最大生命：`80 + 体质×12 + 等级×10`
- 攻击：`武器攻击 + 力量×2 + 等级 + 技能修正`
- 防御：`护甲防御 + 体质 + floor(等级/2) + 状态修正`
- 最终伤害：`max(1, 攻击×随机系数0.85—1.15 - 防御×0.6)`
- 暴击：伤害×1.5；基础暴击率按核心数值文档计算。
- 命中默认100%，只有致盲、闪避姿态、特定技能或环境规则引入未命中。

## 5. 岚音同行AI

岚音自动行动。玩家每回合可在自己行动前设置一次倾向，不额外消耗行动：

- 进攻：优先攻击被标记或低生命敌人；
- 防守：优先保护枫月、打断危险动作；
- 支援：优先标记弱点、使用有限恢复或控制。

倾向只改变AI权重，不保证具体动作。岚音无法行动时，系统必须说明原因。其他临时同伴沿用同一接口，不新增完整手操系统。

## 6. 状态效果

| 状态 | 规则 | 叠加 |
|---|---|---|
| `bleed`流血 | 回合末失去固定生命，持续1—3回合 | 刷新时长，不叠伤害 |
| `poison`中毒 | 回合末按最大生命比例失去生命 | 最多2层 |
| `stun`眩晕 | 跳过一次行动 | 不连续叠加，首领有抗性 |
| `blind`致盲 | 普通攻击和指定技能命中率下降 | 刷新时长 |
| `exposed`破绽 | 下一次受到的伤害提高25% | 消耗后移除 |
| `guarded`防御 | 直接伤害降低40% | 不叠加 |
| `marked`标记 | 岚音和特定技能获得命中/伤害加成 | 单一来源刷新 |
| `weakened`虚弱 | 攻击降低20% | 刷新时长 |

所有状态必须在状态效果数据表登记持续时间、结算时点、免疫、清除方式和UI文本。

## 7. 撤退

撤退成功率：`50% + (玩家敏捷-敌方最高敏捷)×5% + 道具/状态修正 - 首领修正25%`，最终限制在10%—90%。失败消耗行动。撤退成功不得等同任务失败，必须进入已登记的`continuation_id`，产生时间、资源、敌人状态或NPC态度变化。

## 8. 调查与检定

普通调查采用条件满足即成功，例如拥有洞察术、已取得证据、在正确地点或带有相应工具。不得为日常交互滥用随机骰。

重要检定采用可保存随机种子：`属性 + 技能加值 + 装备/状态修正 + d20`。建议难度：15简单、20标准、25困难。结果分为：

- 成功：达到或超过难度；
- 部分成功：低于难度1—4点，获得目标但付出资源、时间或关系代价；
- 失败续接：低于难度5点以上，进入替代版本，不直接卡死主线。

读档必须恢复同一随机种子，禁止通过重复读档立即刷新同一次检定结果；玩家改变行动、装备或准备后可形成新的检定上下文。

## 9. 首领模板

每个首领只允许一个独特核心机制和2—3个阶段。模板必须包含：阶段触发、危险动作、观察提示、可利用机制、失败续接、撤退规则、掉落保底。阶段通常按生命阈值或一次性事件触发，不同时堆叠多个小游戏。

示例：黄金战熊核心机制为“大地震击前摇与地形规避”，三个阶段分别为沉睡突袭、常态战斗、低生命黄金形态。

## 10. 敌人AI

普通敌人使用权重行为表，不写任意脚本：攻击最低生命、攻击最高威胁、施加状态、防御、逃跑。首领可在权重表上增加阶段动作序列。AI使用的每个动作必须来自技能数据表，任务文本不得临时创造代码中不存在的技能。
