# Windows UI/UX规范

## 1. 窗口与分辨率

目标分辨率1920×1080，最低可用1280×720；窗口可缩放、可全屏。当前只保证Windows鼠标与键盘。UI使用可伸缩容器和最小尺寸，不针对某个分辨率写死位置；立绘裁切不能遮挡姓名、正文或选项。

## 2. 三模式主界面

### 2.1 探索阅读模式

用于地点叙事、系统提示、任务说明、采集和战斗日志。

- 顶栏：地点、时间片、等级、生命、金币。
- 主文本区：约占内容宽度68%—72%，显示叙事、系统提示和战斗日志。
- 右侧状态栏：约占25%—30%，显示当前目标、关键状态、同行者与快捷入口。
- 选择区：主文本下方，通常2—5项；支持数字键1—5。
- 底部：历史、自动播放、文本速度、保存和菜单。
- 本模式默认不显示大立绘，必要时仅显示小头像或同行标记。

### 2.2 NPC对话模式

进入具名NPC对白节点时自动切换。

- 左侧立绘区：约占28%—32%，显示当前说话NPC透明背景半身立绘。
- 中部对白区：约占48%—54%，包含姓名牌、对白、动作描写与系统补充。
- 右侧精简栏：约占16%—20%，只保留当前目标、关系提示和同行状态。
- 底部选择区：显示2—5项选择；没有选择时显示继续提示。
- 多人对话只显示当前说话人。切换说话人时以120—180毫秒淡入淡出替换，不同时展示多个大立绘。
- 1280×720时右侧栏允许折叠为图标按钮，优先保证正文、姓名和选择完整显示。

### 2.3 文本战斗模式

显示敌方状态、战斗日志、枫月与同行者状态、六类动作、技能冷却、消耗品和观察到的首领机制。敌方不使用大立绘或动画。详细布局、键位和功能页见`21A_战斗UI与功能页线框规范.md`。

## 3. 立绘显示规则

- 正式对白节点存在`speaker_id`时自动显示该NPC的默认或指定表情。
- 叙事、系统、主角内心和战斗节点默认隐藏大立绘，除非节点明确要求保持上一立绘。
- 立绘以人物眼睛和脸部为视觉中心，不能裁掉头顶、下巴或主要职业标识。
- 缺少立绘或指定表情时显示统一灰色剪影，姓名和对白照常显示，并写入错误日志。
- 表情切换不自动推进文本；快速点击不能导致立绘与对白错位。
- 存档读取到对话节点时，根据节点重新构建立绘，不把纹理本身写入存档。

## 4. 功能页

任务、背包、关系、地点和设置使用同一套标签页或弹层，不创建大量独立窗口。关系页可显示NPC小头像、姓名、关系阶段和最近事件，但不直接显示隐藏数值。关键操作两步内可达。

## 5. 键位

- `Space/Enter`：推进文本或确认。
- `1—5`：选择选项。
- `J`：任务；`I`：背包；`R`：关系；`M`：地点；`L`：历史。
- `F5`：快速保存；`F9`：快速读取；`Esc`：返回/菜单。
- 所有快捷键可在设置中查看；不要求用户记忆才能通关。

## 6. 阅读体验

默认正文字号22px，支持18—32px；行距1.45—1.7；单行长度控制在28—42个汉字附近。支持关闭逐字显示、调节速度、自动滚动、历史回看和高对比模式。选择出现后不得自动倒计时。字号调到32px时，对话区可以增加纵向滚动，但不能压住选项。

## 7. 反馈与无障碍

状态变化用短提示，例如：`【岚音似乎更信任你的判断。】`，不直接显示`trust +2`。战斗数值可详细显示。保存成功、失败、数据错误、立绘缺失和奖励入保管箱必须有明确提示。按钮、焦点、禁用和选中状态必须可区分；不得把关键信息只放在颜色或表情中。

## 8. 地点色调卡与音频

固定8张地点色调卡、10—15个短音效和3首循环音乐。背景只提供地点氛围，不承载关键线索；关键线索必须写入正文或系统提示。无配音、无动态音乐、无环境声分层。

## 9. 自由读档

允许自由手动存档和读取。关键不可逆选择前自动创建独立选择档，不覆盖手动档。
