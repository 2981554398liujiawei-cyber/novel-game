# 战斗UI与功能页线框规范

## 1. 第三种主界面模式：战斗

除探索阅读模式和人物对话模式外，增加战斗模式。战斗不加载独立3D或格子地图，使用地点色调卡、敌我状态卡和文本日志。

1920×1080建议布局：

- 左侧28%：敌方列表、当前目标、生命、状态和危险动作提示；
- 中部50%：战斗叙事、最近6—10条日志、观察结果和阶段提示；
- 右侧22%：枫月与同行者状态、技能冷却、消耗品快捷栏；
- 底部：六类动作按钮和数字键提示。

1280×720时右侧同行者面板折叠到中部顶部，日志最多显示6条；任何分辨率下都不得遮挡动作按钮、生命值和首领机制提示。

## 2. 战斗交互

数字键1—6对应普通攻击、防御、技能、物品、观察、撤退。技能和物品进入二级列表，`Esc`返回一级。玩家可用`Q/E`切换目标，`Tab`查看完整状态说明。岚音倾向使用三个小按钮或快捷键`Z/X/C`，设置不消耗行动。

危险动作必须同时使用文字和图标，不只靠颜色。观察成功后把机制说明固定在战斗日志上方，避免玩家翻历史才能操作。

## 3. 功能页

任务、背包、角色、关系、地点、历史、存档和设置采用统一标签页，不打开多个系统窗口。功能页打开时暂停文本推进和战斗计时；当前版本没有实时战斗，因此不存在后台敌人行动。

- 任务页：主线、支线、关系任务，显示当前目标、已知线索、失败续接和回访，不显示隐藏条件。
- 背包页：分类、装备、任务保管箱、比较信息和使用限制。
- 角色页：属性、经验、装备槽、技能装配、状态说明。
- 关系页：用阶段文字描述，不显示精确好感数值。
- 地点页：节点连接、可用事件和旅行时间，不显示自由行走地图。
- 历史页：按旁白、对白、系统、战斗筛选。

## 4. 背景与声音范围

使用8张简洁地点色调卡，对应回响祭坛、村庄广场、韩石铁铺、苏芷药庐、北药田、西北狼径、悬崖藏宝洞、天枢神泉。色调卡可以是模糊背景、纹理或低细节场景，不制作精细场景插画。

音频硬范围：10—15个短音效，覆盖界面确认、选项、任务更新、奖励、普通命中、暴击、受伤、警示、首领阶段、存档成功和错误；3首循环音乐分别用于村庄、野外、首领。无配音、无分层环境声、无动态音乐系统。

## 5. 读档与关键选择

允许玩家自由存读档查看其他选择。重要不可逆选择前自动建立“关键选择存档”，不覆盖最近的手动档。读档后恢复当前节点、立绘、背景、音乐、随机种子和战斗状态。
