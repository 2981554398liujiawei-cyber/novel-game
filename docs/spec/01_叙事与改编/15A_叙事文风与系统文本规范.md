# 叙事文风与系统文本规范

## 1. 视角与时态

正文采用第二人称近距离叙事，以“你”指代枫月；主角明确发言显示姓名“枫月”。旁白只描述枫月当前能够观察、听见、触碰、推断或回忆的内容，不进入其他角色内心。需要展示他人想法时，通过动作、停顿、语气、证据或后续行为表现。

默认使用现在时或紧邻行动的过去完成描述，单个任务内不得频繁切换。禁止突然出现全知旁白解释幕后真相。

## 2. 枫月的认知限制

枫月没玩过《王者》，不得使用攻略式判断。允许说“这像是首领的前摇”“系统把它标成任务物品”，不允许说“按照原剧情下一步会出现黄金战熊”。他可依据常见游戏经验猜测，但猜测必须标明不确定性，并允许猜错。

## 3. 角色语言指纹

- 枫月：句子简洁，先问关键问题；紧张时减少修饰，确认安全后才会调侃。
- 岚音：观察具体、动作导向，较少空泛安慰；关系升温后语气会柔和，但不突然撒娇。
- 顾长川：缓慢、留白、常从村庄责任出发；不会把世界观一次讲完。
- 韩石：直接、实用，用火候、金属和手艺作比喻。
- 苏芷：精确、克制，用药性、伤势和剂量判断问题。
- 王五：外放但不轻佻，战斗经验通过短句和警告体现。
- 玩家类NPC：可使用玩家术语，但不能所有人都说同一种网络腔；现实话题只在动机成立时出现。

每名具名角色的人物圣经必须登记常用词、句长、礼貌程度、紧张表现、说谎表现和面对不同关系对象的语气变化。

## 4. 信息呈现

重要设定不得只靠NPC站桩说明。优先顺序：可观察证据 → 人物冲突或误会 → 系统提示 → NPC解释 → 日志整理。系统提示只确认系统能够知道的信息，不替编剧宣布道德结论。

## 5. 系统文本格式

- 普通系统提示：`【获得：止血草×3】`
- 洞察信息：`【洞察：夹齿上的蓝粉只沾在金属内侧。】`
- 任务更新：`【任务更新：兔王不该出现】`
- 状态变化：用自然语言，如`【岚音开始相信你的判断。】`，默认不显示`信任+1`。
- 调试构建可显示精确变量，正式构建隐藏。
- 错误信息必须可理解，如`【资源缺失：已使用通用立绘。】`，不得只显示异常堆栈。

系统NPC和玩家类NPC都能看到系统，但魔兽不能。玩家可见对白不得出现制作术语`玩家类NPC`、`系统NPC`、`node_id`、`state_key`。

## 6. 长度与节奏

单段建议40—120字，连续说明最多3段；正式对白通常15—80字。长信息必须拆成可回看日志，不用一段千字说明。每个场景应在阅读、行动、回应之间交替，连续十个节点不得全部为被动听讲。

## 7. 标点与称谓

统一使用中文全角标点。对白数据的`text`不强制添加外层引号，由UI姓名牌识别说话人；动作可写在独立旁白节点或`gesture`字段。称谓必须符合关系阶段，不得在信任尚低时提前使用亲密称呼。
