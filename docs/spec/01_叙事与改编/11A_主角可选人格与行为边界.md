# 主角可选人格与行为边界

## 1. 固定人格核心

枫月24岁，现实本名与游戏名相同。他对《王者》的具体内容几乎不了解，进入世界后的基本反应是先确认环境、寻找规则、保护自身，再判断是否信任他人。他不是预设圣人，也不是冷血工具人。无论玩家怎样选择，必须保留以下连续性：

- 对未知保持警觉，不会无证据断言真相；
- 重视自身生存，但不会为了娱乐伤害无辜；
- 能接受利益、稀有掉落和力量诱惑，也能意识到行为会产生后果；
- 面对他人的善意、伤势或恐惧会产生真实反应；
- 不会进行性胁迫、虐待、纯粹毁灭或无理由背叛；
- 选择隐瞒、私藏或拒绝救援时，应有现实动机，而不是突然变成另一个人。

## 2. 允许的玩家取向

| 取向 | 合法表现 | 不合法写法 |
|---|---|---|
| 善良 | 优先救人、保护生态、分享资源 | 无条件相信所有人、主动牺牲到不顾逻辑 |
| 谨慎 | 先调查、保留退路、延后承诺 | 永远不行动、用谨慎拖死剧情 |
| 功利 | 谈报酬、争取战利品、选择高收益方案 | 为微小利益虐杀无辜或毁灭村庄 |
| 怀疑 | 质疑NPC、验证证据、不透露现实信息 | 没有证据就认定所有人是程序或骗子 |
| 冒险 | 挑战首领、探索隐藏路线、承担风险 | 明知会无意义害死他人仍冲动行事 |
| 冷淡 | 拒绝亲近、只完成任务、不回应暧昧 | 羞辱他人、以残酷为乐 |
| 隐瞒 | 不说明现实、私藏部分信息或物品 | 通过谎言实施性胁迫或蓄意害死无辜 |

## 3. 选项设计字段

每个重要玩家选项必须记录：

- `intent`：玩家表达的态度或行动目的；
- `protagonist_boundary`：`allowed`或`blocked`；正式数据中只允许`allowed`；
- `visible_risk`：选择前玩家能够合理预见的风险；
- `consequence_summary`：不剧透隐藏真相的直接后果提示；
- `hidden_consequence`：供编剧和程序引用的延迟后果，不直接显示；
- `relationship_tone`：信任、疏离、功利、试探、关怀等标签；
- `goto`与`effects`：明确的程序跳转和状态写入。

## 4. 不能只用“接受/拒绝”

重要选择至少体现行动或价值差异，例如：先救伤者、先确认敌情、要求报酬、保留证据、相信岚音、独自承担、暂时撤退。接受和拒绝可以存在，但不能成为所有任务的唯一分支。

每个选项必须在一至两个节点内得到即时回应。即使后续汇流，NPC态度、资源、证据、战斗开局、时间片或回访文本至少有一项不同。

## 5. 越界选项处理

编剧AI不得生成以下选项：主动虐杀非敌对无辜者、性胁迫、以折磨为乐、纯粹毁灭村庄、无动机杀害队友、对已投降者继续施暴。若剧情需要表现诱惑或恶意，应让枫月拒绝、制止、绕开或承担被迫局面，而不是把越界行为交给玩家执行。
