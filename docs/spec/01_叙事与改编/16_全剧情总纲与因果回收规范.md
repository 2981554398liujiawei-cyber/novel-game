# 全剧情总纲与因果回收规范

## 1. 目的

全剧情总纲用于回答“为什么发生、玩家能怎样介入、失败后世界如何继续、伏笔何时回收”，不能只列任务名称或事件顺序。总纲通过前，任何AI或开发人员不得直接批量生成正式任务；否则容易得到一串看似完整、实际没有因果和回访的任务菜单。

## 2. 每一幕的最低深度

新手村篇章分为四幕。每一幕必须明确填写：前因、主要矛盾、阻力、枫月的阶段目标、玩家可选行动顺序、关键发现、误导信息、中点反转、最低谷、高潮、结局、遗留问题、后续世界变化。缺少任一项时，该幕只能标记为`outline_draft`。

| 字段 | 必须回答的问题 |
|---|---|
| 前因 | 什么既有状态导致本幕发生，而不是“系统突然刷任务”？ |
| 主要矛盾 | 枫月想完成什么，谁或什么阻止他？ |
| 阻力 | 人物立场、证据不足、资源、战斗、时间或规则限制分别是什么？ |
| 阶段目标 | 玩家此刻可以理解并执行的目标是什么？ |
| 可选顺序 | 哪些目标强顺序，哪些可并行，改变顺序会怎样？ |
| 关键发现 | 本幕结束后玩家必定知道什么？ |
| 误导信息 | 哪个表面解释会被后续证据修正？ |
| 中点反转 | 玩家对问题的理解何时改变？ |
| 最低谷 | 玩家付出代价、判断失误或暂时失败的节点是什么？ |
| 高潮 | 哪个选择或行动集中兑现本幕矛盾？ |
| 结局 | 本幕解决了什么，哪些状态被写入？ |
| 遗留问题 | 哪个问题必须带入下一幕？ |
| 世界变化 | NPC、地点、价格、传闻、敌人或可用路线发生什么变化？ |

## 3. 当前四幕定义

### 第一幕：陌生规则

- 前因：枫月在回响祭坛醒来，系统将他识别为玩家，却没有退出入口。
- 主要矛盾：他必须先取得村庄信任和生存资源，才能调查自己为何被困。
- 阻力：身份无法证明；顾长川有所隐瞒；教程界面看似正常，容易让枫月误判一切只是异常登录。
- 玩家顺序：`NV_MAIN_001`强顺序；`NV_MAIN_002`内铁匠、药师和村长委托可并行，至少完成两项才触发药田事件。
- 关键发现：居民会受伤、物品具有真实重量、退出通道锁定。
- 误导：系统仍显示等级和任务，制造“只是沉浸式故障”的假象。
- 中点反转：顾长川能察觉枫月的系统辉光存在异常偏移。
- 最低谷：枫月确认无法联系现实，也无法靠教程直接离开。
- 高潮：玩家决定向顾长川坦白、隐瞒或反问，并主动接受三份委托。
- 结局与变化：获得初始装备和洞察术；村庄对枫月形成第一印象；北药田异常被激活。
- 遗留：谁锁住返回通道，顾长川为何能识别系统辉光异常？

### 第二幕：幸运不是答案

- 前因：捕兽者利用蓝色药粉和陷阱驱赶兔群，药田面临破坏。
- 主要矛盾：玩家要在保护药田、调查幕后与争取稀有掉落之间取舍。
- 阻力：兔群混乱、证据可能被毁、岚音不信任只凭运气的人。
- 玩家顺序：先调查、先驱散或直接狩猎均可；路线在结算处汇流，但生态、药价、证据和关系状态必须不同。
- 关键发现：魔物异常可能由人为行为和神泉失衡共同放大。
- 误导：表面上是“兔王袭村”，实际是人为驱赶与活体收购。
- 中点反转：硬底靴印和蓝粉证明怪物不是唯一敌人。
- 最低谷：战败后药田继续受损，玩家在苏芷药棚醒来。
- 高潮：保护兔群、揭露捕兽者或猎杀兔王。
- 结局与变化：兔群生态、药价、捕兽者调查和岚音信任写入状态。
- 遗留：谁在收购异常魔物，神泉失衡与进化有什么关系？

### 第三幕：规则中的人心

- 前因：异常向狼径蔓延，天火冷魂被刺猬王困住，风云三人组追逐稀有战利品。
- 主要矛盾：枫月要证明自己是否把他人当作真实的人，而非可刷新资源。
- 阻力：救援会消耗资源；山狼王与风云三人组形成多方冲突；藏宝图信息可能是诱饵。
- 玩家顺序：救援、观察或绕开天火冷魂；随后可威慑、智取或正面应对风云三人组。
- 关键发现：排行榜与掉落会诱导冒险者把风险转嫁给别人；藏宝图指向世界边界异常。
- 误导：藏宝图像普通奖励，实际带出“外部坐标不可用”。
- 中点反转：山狼王不是唯一危险，抢夺者会利用战斗结果。
- 最低谷：伙伴或玩家战败，战利品被夺走一部分，必须使用替代路线追回线索。
- 高潮：在山狼王与抢夺者之间选择战术和战利品分配。
- 结局与变化：天火冷魂关系、风云三人组态度、宝藏入口和岚音评价发生变化。
- 遗留：外部坐标为何失效，神泉是否是世界稳定锚点？

### 第四幕：神泉试炼

- 前因：藏宝洞和王五的证词共同指向天枢神泉，黄金战熊占据核心区域。
- 主要矛盾：枫月要夺回并净化神泉，同时决定是否私藏力量、信任同伴和承担后果。
- 阻力：黄金战熊多阶段战斗、未净化泉水诱惑、村庄对数量和用途有明确要求。
- 玩家顺序：战术准备可并行；战熊战、取水、检查或私藏、净化与离村按强顺序推进。
- 关键发现：神泉是新手村保护结界的锚点；枫月的返回通道与世界出口不是同一扇门。
- 误导：净化泉水看似能解决一切，实际只能打开前往更大世界的出口，不能直接返回现实。
- 中点反转：战败不会重置世界，王五救援会留下受伤版战熊和资源代价。
- 最低谷：枫月发现即使恢复神泉，`RETURN CHANNEL`仍为锁定。
- 高潮：净化九坛泉水、处理私藏或完整交付，并与岚音完成关系结算。
- 结局与变化：村庄保护恢复，离村门开启，枫月决定继续寻找回归方法。
- 遗留：返回通道由谁控制，岚音是否会在后续区域同行？

## 4. 任务线必须互相引用

当前区域必须至少包含七类互相引用的内容：主线、区域公共线、伙伴线、外来冒险者/捕兽者线、村庄经济反馈、装备与能力成长、隐藏异常与结算回访。它们不要求各自形成独立菜单，但必须通过状态相互影响。例如兔群生态改变苏芷药价，捕兽者证据改变顾长川回访，岚音信任改变神泉同行版本，藏宝洞异常强化退出通道伏笔。

禁止创建“写入后从未读取”的永久状态。每个永久状态必须登记首次写入、至少一次中期读取和章节结算读取；若没有后续差异，应改为局部变量或删除。

## 5. 强顺序、并行和状态版本

- 强顺序：穿越开场、兔群事件、狼径、藏宝洞、神泉、取水、净化与离村的主因果顺序不可颠倒。
- 可并行：三份委托、部分资源收集和任务后回访。
- 状态版本：同一地点至少区分首次、任务进行中、战败后、任务完成后四种进入文本。
- 临时关闭：战败、夜晚或NPC受伤可以暂时关闭某入口，但必须提供等待、替代NPC、回访或救援路线，不得永久卡死主线。
- 重要剧情不可因错过一个时间片永久消失；应转为迟到版、战后调查版或资源代价版。

## 7. 总纲验收

审阅者只看总纲，就应能说清每幕为什么发生、玩家能改变什么、失败后如何继续、状态在哪里回收。只有事件列表、BOSS列表或奖励列表的文件必须标记为`DRAFT`，不得作为AI生成正式任务的输入。
