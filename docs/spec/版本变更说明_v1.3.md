# v1.3版本变更说明

- 冻结索达公司最新游戏、枫月没玩过游戏且使用现实本名。
- 冒险者统一为现实玩家类NPC，可复活、知道现实、无法退出；系统NPC能看系统但无现实记忆；魔兽看不到系统。
- 依据原著规则区分玩家复活与系统NPC死亡；当前新手村禁止新增具名系统NPC永久死亡。
- 冻结枫月可选人格与禁止行为。
- 新增完整战斗、成长经济、战斗UI、Windows运行和内容权威源规范。
- 音画范围固定为8地点色调卡、10—15音效、3音乐、9具名+3通用人物立绘。
- 总文本目标38,000—50,000字，目标时长保持3—4.5小时。
- 修复正式状态默认值、主线003前置、苏芷立绘show、双内容状态、任务结构化和校验器语义缺口。
- 补充GameState运行时写入来源保护、原子批量修改、条件操作符和越界拒绝规则；未新增正式状态变量。
- 补充最小StoryRunner节点、信号、GameState委托、错误处理、自动跳转保护和运行位置契约；未导入正式剧情。
- 补充SaveManager五类固定槽位、Windows用户目录、原子写入、覆盖备份、显式备份恢复、版本占位与GameState/StoryRunner原子往返契约；首次启用开发存档`save_version=1`，不兼容此前未落地的旧开发存档结构，未增加正式状态或剧情内容。
- 补充InventoryManager固定容量、独立任务物品区、关键奖励保管箱、六装备槽、消耗品效果、任务奖励接入与版本化存档契约；物品运行字段采用向后兼容的可选`runtime`区块，存档Schema升级至`1.1.0`，未新增正式物品ID、状态变量或剧情内容。
- 补充CombatRunner数据契约次版本：战斗、敌人与技能Schema增加向后兼容的可选`runtime`区块，登记集中战斗规则、状态解释、唯一单位实例、类型化阶段、观察、撤退、结算及数据驱动AI；1.3正式条目继续兼容，新增合成数据仅位于测试fixtures，未新增正式战斗、敌人、技能或平衡内容。
- 补充战斗物品与存档边界：物品`runtime`增加可选`combat_effects`，由CombatRunner预演本场效果后通过InventoryManager原子消费；活跃战斗统一禁止保存、读取和备份恢复，CombatRunner必须绑定SaveManager并成功创建战前自动检查点才可启动，未提升`save_version`且未加入正式物品。
