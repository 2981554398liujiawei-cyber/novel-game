## 2. 变更规则

任何新增功能必须说明它替代或删除了什么。以下变更必须提交“范围变更单”：新增系统、突破上限、增加后续区域、增加第二名可发展关系角色、增加动态立绘或动画系统、修改终局契约、修改主角姓名、修改存档格式、引入第三方插件或在线服务。

范围变更单至少包含：变更原因、玩家价值、开发成本、美术成本、测试成本、对已有数据与存档的影响、被删除内容、验收标准和回滚方案。没有明确删减项的新增请求默认不进入当前版本。

## 3. 立绘范围控制

每个具名NPC都有独立半身立绘，但不等于每人都配置大量表情。A级角色最多4个表情，B级角色最多2个表情，C级角色只做1个默认表情。新增表情必须说明它服务的剧情节点；仅用于“更好看”的重复表情不进入当前版本。有对白但未具名的角色只使用3套正式通用立绘：无名男村民、无名女村民、蒙面冒险者。纯背景人群不显示立绘；统一缺图剪影只用于资源异常。

## 4. 优先级

P0：启动、内容加载、主线、对话与立绘正确显示、战斗、存档、失败续接、数据校验。P1：关系反馈、日志、720p/1080p适配、可用性、平衡。P2：额外表情、装饰性转场、环境音效。P3：后续区域构想。任何P2/P3事项不得阻塞P0/P1。
