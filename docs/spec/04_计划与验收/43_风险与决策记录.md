# 风险与决策记录

| 风险 | 表现 | 已冻结对策 |
|---|---|---|
| 主角全知 | AI借原著让枫月知道攻略 | 枫月没玩过游戏；判断必须有证据 |
| 冒险者身份冲突 | 一会儿像程序，一会儿像现实玩家 | 全部冒险者为玩家类NPC，来自现实、可复活、无法退出 |
| 系统NPC知识越界 | 村民谈索达公司或现实设备 | 系统NPC可见系统但无现实知识 |
| NPC死亡破坏后续 | AI在失败分支写死任务角色 | 当前系统NPC不永久死亡；未来按原著命运映射审批 |
| 主角人格碎裂 | 选项包含残虐或性胁迫 | 使用主角行为边界和选项字段校验 |
| 三份委托前置错误 | 样例跳过主线002 | 记录完成数，≥2才解锁主线003 |
| 测试污染正式数据 | 默认状态写成已完成 | 正式默认与fixtures分离，manifest禁载fixtures |
| 双状态体系 | approved与writing_status冲突 | 只保留content_status |
| 立绘keep空屏 | 前序节点未显示说话人 | 校验立绘状态流，无保证时必须show |
| Schema名义完整 | 文档承诺多于校验器 | 补全Schema和跨文件语义检查 |
| 战斗规则漂移 | 开发AI自行补MP、行动序 | 固定无MP、敏捷顺序、冷却/每战次数和首领模板 |
| 美术范围膨胀 | 所有路人独立立绘 | 9具名+3通用+1缺图，背景8张低细节 |
| 文本时长失控 | 单任务达标但总量过大 | 总字数38,000—50,000并以3—4.5小时实测裁剪 |
| Markdown/JSON漂移 | 编剧和开发改不同版本 | 分阶段单一权威源，JSON生成审阅稿 |
