# 剧情内容验收与DONE定义

## 1. 自动门槛

正式任务必须通过：Schema、重复ID、引用、状态登记、节点可达性、完成出口、占位词、正文字符数、正式话轮数、场景数、选择节点数、测试用例数、对白表情和即时回应检查。自动门槛只负责挡住明显骨架，不能代替人工审稿。

## 2. 人工剧本验收

审阅者在不知道原著的情况下，应能回答：人物为何出现、枫月当前目标、阻力是什么、每个选择做了什么、NPC如何回应、失败如何继续、任务结束后哪里改变。若需要作者口头补充，任务不合格。

重点检查：人物语言是否可区分；信息是否通过冲突与证据呈现；分支是否改变过程或状态；战斗前后是否有剧情连接；失败是否产生可理解代价；永久状态是否在后续读取；关系内容是否自愿且可拒绝。

## 3. 运行验收

逐条按任务测试用例运行，确认文本、立绘、表情、选择条件、状态写入、奖励、失败重返、存档恢复和任务后反馈与剧本一致。任何脚本为迁就代码而删减关键对白或合并后果，视为严重缺陷。

## 4. 状态定义

- DRAFT：只有总纲、设计卡、部分场景或示范台词。
- SCRIPT COMPLETE：完整正文与分支已写完，尚未转数据。
- DATA READY：JSON校验通过，尚未完整运行。
- IMPLEMENTED：已接入工程，尚未完成全部测试。
- DONE：自动校验、人工通读、运行测试和制作方签收全部通过。

代码提交、字数、图片数量或“主路线能跑”均不能单独构成DONE。
