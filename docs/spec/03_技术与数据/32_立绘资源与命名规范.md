# 立绘资源与命名规范

## 1. 资产规格

- 类型：透明背景半身立绘，正面或轻微侧身。
- 母版建议：1536×2048像素或同等3:4比例；人物占画布高度约85%—95%。
- 文件格式：PNG，RGBA透明通道；不带背景、文字、边框、水印或阴影底板。
- 安全区：头顶保留约5%—8%，脸部位于画布上方30%—40%区域，主要职业道具不能完全被裁掉。
- 方向：人物视线和身体可略向画面中央，避免持续向屏幕外侧看。
- 色彩：统一光源和饱和度；不得每个角色使用完全不同的渲染风格。

## 2. 命名和目录

```text
assets/portraits/nv7/<npc_slug>/<npc_slug>__<expression>.png
```

示例：`assets/portraits/nv7/lanyin/lanyin__serious.png`。`npc_slug`和`expression`只使用小写英文字母、数字和下划线。资源移动或改名必须同步更新NPC数据；已进入正式构建的表情ID不得更改。

## 3. 角色分级

- A级：主关系角色，最多4个表情。
- B级：主线与功能角色，最多2个表情。
- C级：次要冲突角色，1个默认表情。

表情数量控制的是图片数量，不限制对白丰富度。新表情必须服务明确剧情节点，并经过资产清单批准。

## 4. 制作与接入流程

人物圣经冻结 → 生成或绘制默认立绘 → 检查年龄感、服装、姿势和画风 → 制作表情变体 → 去除背景并统一画布 → 按规则命名 → 登记到NPC数据 → 运行Schema和资源检查 → 在720p/1080p实机检查裁切 → 标记为`approved`。

## 5. 禁止事项

不使用带背景的成品插画直接裁切；不在表情变体中改变服装、发型、体型或年龄；不使用低清放大图；不以身体暴露表现关系升级；不把名字或对白写进图片；不为匿名路人无限新增独立立绘。

## 6. 缺图资源

统一缺图文件：`assets/portraits/fallback/npc_missing.png`。剪影应中性、可识别为人物且不暗示具体性别或职业，界面同时显示真实NPC姓名。缺图只用于开发期和异常回退，候选构建中具名NPC缺图属于严重缺陷。
