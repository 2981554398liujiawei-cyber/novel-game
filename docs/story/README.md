# 剧情包接入说明

本目录把制作方提供的剧情材料纳入仓库治理，但不把所有剧情文档一视同仁。

## 当前层级

| 目录 | 用途 | 当前能否直接实现 |
|---|---|---|
| `active/` | 第七新手村当前剧情源 | 不能直接转生产JSON，需逐任务达到`complete_script/data_ready` |
| `roadmap/` | 全游戏后续篇章路线 | 否，只做未来规划 |
| `reference/` | 樱花岛、旧开放世界设定等参考 | 否，存在旧机制与旧世界规则 |

## 当前新手村剧情源

当前剧情母稿覆盖8个主线、可选剧情、主要分支、失败续接与篇章结尾。它是“当前篇章内容源”，优先用于纠正旧规格中的具体剧情差异，但仍需按`docs/spec/01_叙事与改编/17_完整任务剧本与对白数据规范.md`拆成可执行完整剧本后，才能进入`content/quests/`。

## 重要冲突已处理

旧规格曾把`NV_MAIN_002《三份委托》`写成“黄金铁锤、黄金药鼎、二十条虎尾”。当前剧情母稿明确为“韩石的试刃、苏芷的药篮、顾长川的界石”，本生产包已统一为后者。

旧全剧情任务树和旧开放世界设定仍保留为未来参考，但其中的自由移动地图、7×5战棋、模拟玩家、大规模后台世界等设计不得反向进入当前项目。

## 原始剧情附件

制作方提供的原始剧情文件按`source_manifest.json`登记。当前项目结构以这些文件名和版本为来源标识；运行时不会读取DOCX/Markdown剧情源。
