# docs/story/AGENTS.md

本目录存放剧情源、剧情路线草案与未来参考材料。除根 `AGENTS.md` 外，处理本目录时必须遵守以下规则。

## 1. 目录状态

- `active/`：当前第七新手村剧情源。允许用于拆任务和写完整剧本，但母稿本身不是运行时数据。
- `roadmap/`：后续区域任务树。只做长期规划，不得触发当前开发范围扩大。
- `reference/`：未来区域或旧设计参考。可能包含已废弃机制，不能直接转成实现需求。

## 2. 当前剧情正典边界

- 当前篇章剧情顺序以 `active/00_第七新手村剧情源接入索引.md` 和制作方提供的《王者》第七新手村完整剧情母稿为准。
- 当前三份委托是：韩石的试刃、苏芷的药篮、顾长川的界石。
- 旧文件里的“黄金铁锤、黄金药鼎、二十条虎尾作为三份委托”不再作为当前实现依据。
- 旧任务树中的“模拟玩家”“7×5战棋”“开放世界实时模拟”等表述均属于旧产品形态，不得覆盖当前规格。

## 3. 禁止自动补写

- 不得把剧情母稿自动升级为 `complete_script`。
- 不得根据任务树的“对白钩子”扩写正式对白。
- 不得为了让JSON通过而发明人物动机、关键分支、失败结果或世界状态。
- 缺口使用 `CONTENT_MISSING` 或 `STORY_NOT_DATA_READY` 报告。

## 4. 生产流程

剧情源 → 16字段任务剧本 → 人工审阅 → `complete_script` → 冻结ID/状态/测试 → `data_ready` → 转JSON → 校验 → 实现 → `verified`。

剧情源文件本身不得由Codex在实现任务中顺手改写；需要改剧情时必须单独任务、单独审阅。
