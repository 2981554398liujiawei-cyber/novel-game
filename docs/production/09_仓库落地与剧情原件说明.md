# 仓库落地与剧情原件说明

## 1. 目标仓库

- 仓库：`2981554398liujiawei-cyber/novel-game`
- 默认分支：`main`
- 当前整理时仓库状态：空仓库，可直接将本包内容作为首个工程基线提交。

## 2. 本包的剧情接入方式

剧情被拆成三层：

1. `docs/story/active/`：当前第七新手村剧情源的接入规则与索引。
2. `docs/story/roadmap/`：后续全剧情路线草案，只规划未来，不扩大当前范围。
3. `docs/story/reference/`：樱花岛与旧世界设定参考，不自动成为当前产品需求。

原始附件应归档到 `docs/story/raw_sources/` 对应目录，并由 `docs/story/source_manifest.json` 登记。原始附件不是运行时数据，不进入 `content/manifest.json`。

## 3. 当前新手村剧情冲突已清理

当前 `NV_MAIN_002《三份委托》`采用剧情母稿版本：

- 韩石的试刃；
- 苏芷的药篮；
- 顾长川的界石。

三项可任意顺序推进，完成任意两项后允许触发 `NV_MAIN_003《兔王不该出现》`，剩余一项之后仍可补做并影响回访。

旧材料中的“黄金铁锤 / 黄金药鼎 / 二十条虎尾”不得继续作为当前三份委托定义。

## 4. 原始附件缺失时的行为

若 `docs/story/raw_sources/` 中没有对应原始文件：

- 可以继续开发不依赖具体剧情正文的技术骨架；
- 可以读取已经冻结的产品、系统、数据和剧情接入规则；
- 不得自行补写正式对白、任务结局或缺失场景；
- 需要核对原文时报告 `CONTENT_MISSING`；
- 已有母稿但未达到任务级 `data_ready` 时报告 `STORY_NOT_DATA_READY`。
