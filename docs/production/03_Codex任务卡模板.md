# Codex 任务卡模板

每次给 Codex 的开发任务至少包含以下字段：

## Goal

一句话说明本任务最终要达成什么。

## Context

列出必须读取的规格、相关模块和现有接口。

## Scope

明确允许修改的目录和功能。

## Out of scope

明确本任务不能顺手做什么。

## Constraints

列出关键技术、产品和数据边界。

## Acceptance criteria

使用可检查的通过条件，不写“效果良好”一类模糊表述。

## Required validation

列出完成前必须运行的命令。

## Report format

最终汇报必须包含：修改文件、主要实现、验证命令及结果、未解决问题。
