# 生产启动说明

本目录负责把已经冻结的设计与剧情源转换为可持续开发流程。

开工顺序：

1. 验证开发环境与仓库结构。
2. 建立验证、测试和CI闭环。
3. 实现 ContentLoader、GameState、StoryRunner接口和三模式UI骨架。
4. 实现存档、任务、战斗、背包与关系模块。
5. 用合成fixture验证完整运行链。
6. 剧情侧把`NV_MAIN_001—003`从母稿拆为16字段`complete_script`并审核到`data_ready`。
7. 使用内容转换工具生成正式任务JSON，再接入垂直切片。
8. 垂直切片签收后冻结存档兼容边界。

剧情母稿已经存在，因此技术开发可以参考其流程设计接口，但任何阶段都不得由Codex自行补写缺失剧情并悄悄进入正式内容。
